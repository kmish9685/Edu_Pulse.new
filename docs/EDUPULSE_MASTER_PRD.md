## EXECUTIVE SUMMARY

### **Product Vision**
EduPulse is an institutional learning intelligence platform that transforms invisible student confusion into actionable teaching insights and measurable learning improvements.

### **The Problem We Solve**
Across Asian classrooms, 30+ million students fail annually because their confusion goes undetected until exams reveal learning gaps. Current methods (verbal questions, end-of-class surveys, facial expressions) are inadequate:
- Only 12% of confused students raise hands (cultural barriers, fear, large class sizes)

### **Database Schema (Prisma Format)**

```prisma
// schema.prisma

model Institution {
  id                String    @id @default(uuid())
  name              String
  location          String
  campusLatitude    Float
  campusLongitude   Float
  campusRadius      Int       @default(500) // meters
  subscriptionTier  String    // "pilot", "school", "enterprise"
  courses           Course[]
  createdAt         DateTime  @default(now())
}

model Course {
  id            String      @id @default(uuid())
  name          String
  code          String
  department    String
  institution   Institution @relation(fields: [institutionId], references: [id])
  institutionId String
  sessions      Session[]
  createdAt     DateTime    @default(now())
}

model Session {
  id              String            @id @default(uuid())
  course          Course            @relation(fields: [courseId], references: [id])
  courseId        String
  teacher         User              @relation(fields: [teacherId], references: [id])
  teacherId       String
  startTime       DateTime
  endTime         DateTime?
  qrCode          String            // Generated QR code URL
  confusionSignals ConfusionSignal[]
  clarityScore    Int?              // 0-100
  createdAt       DateTime          @default(now())
}

model ConfusionSignal {
  id              String    @id @default(uuid())
  session         Session   @relation(fields: [sessionId], references: [id])
  sessionId       String
  deviceId        String    // FingerprintJS device ID (anonymous)
  topic           String?   // Selected topic (optional)
  timestamp       DateTime  @default(now())
  latitude        Float?    // For geofencing validation
  longitude       Float?
  resolved        Boolean   @default(false) // Did confusion decrease after?
}

model Topic {
  id          String   @id @default(uuid())
  name        String   // "Database Normalization - 3NF"
  category    String   // "Database Systems"
  course      String   // Course code
  createdAt   DateTime @default(now())
}

model User {
  id          String    @id @default(uuid())
  email       String    @unique
  password    String    // bcrypt hashed
  name        String
  role        String    // "teacher", "admin"
  institution String
  sessions    Session[]
  createdAt   DateTime  @default(now())
}

model LearningOutcome {
  id          String   @id @default(uuid())
  sessionId   String   // Session where this assessment happened
  courseId    String
  assessmentType String // "quiz", "midterm", "final"
  averageScore Float   // Average score for students in this session
  controlScore Float?  // Average score for control group (if applicable)
  createdAt   DateTime @default(now())
}
```

**Indexing Strategy:**
- `ConfusionSignal.sessionId` (fast aggregation per session)
- `ConfusionSignal.timestamp` (time-series queries)
- `ConfusionSignal.topic` (clustering queries)
- `Session.courseId` (recurring gap detection)
- `User.email` (login lookup)

**Contingency:**
- If <10 faculty by Week 3: Go directly to HODs, get mandate
- If students don't use it: Add incentive (extra credit? lottery?)
- If technical issues scare faculty away: Pause, fix bugs, relaunch

---

### **Risk 3: Competitor Makes Big Move**
**Scenario:** Rank 2 or 3 announces major partnership/funding/feature

**Mitigation:**
- Don't try to match their announcement
- Double down on YOUR differentiation (most improved, fastest execution)
- Use their success to validate market (if they're raising money, market is real)
- Position as complementary (we can integrate with them)

**Contingency:**
- Emphasize PROGRESS over status (we shipped 10 features in 8 weeks, what did they do?)
- Highlight student authenticity (we're not corporate, we're grassroots)

---

### **Risk 4: Data Shows No Learning Improvement**
**Scenario:** Quiz scores don't improve in EduPulse classes vs. control

**Mitigation:**
- Start tracking EARLY (Week 3), so you have time to intervene if numbers are weak
- Ensure control group is truly comparable (same professor, same difficulty)
- Measure multiple outcomes (not just quiz scores - also completion rate, satisfaction)

**Contingency:**
- Pivot to process metrics: "74% participation proves engagement value"
- Use qualitative testimonials: "Students and faculty LOVE it, even if quizzes similar"
- Frame as "early data, needs full semester for conclusive results"

---

### **Risk 5: Judges Don't Understand Technical Sophistication**
**Scenario:** Judges are non-technical, miss innovation in your AI/algorithms

**Mitigation:**
- Avoid jargon in pitch (no "machine learning", say "pattern recognition")
- Use VISUAL demos (show algorithm working, don't explain code)
- Compare to everyday technology: "Like how Netflix recommends shows, we recommend teaching actions"
- Emphasize OUTCOMES ("Teachers make better decisions") over tech ("We use Bayesian inference")

**Contingency:**
- If technical questions arise: "I can dive deeper after the pitch, but the key point is..."
- If they seem confused: Simplify on the fly, use analogies
